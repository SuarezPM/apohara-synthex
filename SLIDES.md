**2 · The drama (May 2026).** Agents are in production scraping the open web — with **no audit trail of what they classified, and no proof of what changed since yesterday**. EU AI Act (Art. 12 logging) + DORA make that a liability. SIEMs see the agent's infra, not the web content it touched. **Triggerware detects deltas; nobody seals them.** And the MCP tooling layer itself now ships CVEs — **CVE-2025-68143** (CVSS 8.8 HIGH, path traversal in `mcp-server-git`, NVD Dec 2025): the attack surface is real.

**3 · Category — v0.6.0 expansion.** Synthex = **the cryptographic proof layer over Bright Data + Triggerware + Cognee**. Not a SIEM, not a scraper, not a governance dashboard: the first pipeline that scrapes → classifies → **signs** → **diffs cross-time with chain-of-custody**.

**4 · Architecture (1 slide, 1 diagram).** `Triggerware → FETCH (Bright Data, 6 surfaces) → FORGE (78-rule DJL + 32-rule prefilter — two layers in main pipeline — plus 25-rule PII filter on monitor/KG-ingest path) [src: src/forge/{djl.js, prefilter.js, pii-filter.js}] → CLASSIFY (AI/ML, 4 lenses ‖, tier selector free/oss/paid) [src: src/classify/{aiml-client.js, tiers.js}] → PROVE (HMAC canonicalize + RFC 3161 TSA with CMS chain verify + multi-page PDF + decisions[] audit trail + delta_chain) [src: src/prove/, src/delta/] → OBSERVE (OTel) → MEMORY (Cognee, opt-in COGNEE_LIVE) → act`. 100% JS (Node 24 LTS), an MCP companion to `brightdata-mcp` — **and** a serverless web app (Vercel) running the same pipeline over Bright Data's REST API, streaming per-stage progress over SSE. Every stage emits OpenTelemetry GenAI spans. Zero runtime deps for the deterministic pre-LLM layer (regex-only, `node:crypto`).
