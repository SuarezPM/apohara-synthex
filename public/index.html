1 OF 3 · TRUST-CRITICAL

The signature is verified link by link, not just stored.

Every RFC 3161 timestamp Synthex emits is walked through the entire DigiCert chain — signer → intermediate → root → pinned anchor — using a custom CMS verifier built on webcrypto.subtle. The verifier returns four reasons, not just true/false. Anchor staleness is no longer indistinguishable from forgery.

  • forgedthe signature math itself fails — bytes were edited or the responder cert is wrong
  • untrusted-anchormath passes but the chain doesn't reach our pins — usually DigiCert rotated, not a forgery
  • chain-incompletethe signer cert isn't in the CMS, or the chain can't be walked
  • nullHMAC-only seal (no TSA token); the verifier short-circuits, no .verify() call

2 OF 3 · CONTINUITY

Each re-scrape carries the previous timestamp's serial.

When a target changes between snapshots, the new Evidence Report carries previous_tsa_serial alongside current_tsa_serial. The chain proves that what changed is what we say changed — and that nothing was inserted between snapshots.

3 OF 3 · FOUR LENSES, ONE SEAL

Every scrape classified four ways, sealed together.

GTM · Finance · Security · Supply-chain run in parallel against the same scraped content. All four findings sealed inside the same Evidence Report, so the seal proves what was found and what the model didn't say. The pipeline runs 78 DJL + 32 prefilter rules pre-LLM (plus a 25-rule PII gate on the monitor / KG-ingest path) — so anything that reaches the LLM is what survived the screen.
