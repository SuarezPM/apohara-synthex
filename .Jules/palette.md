## 2026-05-29 - Adding Form Usability to Live Console\n**Learning:** The live console in public/index.html was not wrapped in a formal <form> tag, which meant natural keyboard navigation (pressing Enter to submit) didn't work. The input also lacked guidance (placeholder) and validation (empty submissions were allowed), leading to a frustrating experience. ## 2026-05-30 - Adding Form Usability to Live Console
**Learning:** The live console in public/index.html was not wrapped in a formal <form> tag, which meant natural keyboard navigation (pressing Enter to submit) didn't correctly trigger error states if the input was empty because the listener only allowed Enter presses if the input was filled. Additionally, the label did not visually distinguish that it was a required field despite the input having the required attribute.
**Action:** When working with custom JS-driven "forms" that don't use standard HTML <form> submission, always ensure keyboard events (keydown Enter) are manually wired to trigger the primary action regardless of input validity, so the error states can be displayed to the user. Always visually denote required fields.
